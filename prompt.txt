Saya sedang mengembangkan backend REST API menggunakan CodeIgniter 3.1.10 untuk sistem ticketing dan monitoring perangkat.

Database MySQL sudah terhubung melalui application/config/database.php.

Saya memiliki beberapa tabel yang saling berhubungan.

Table: devices

- id
- device_name
- device_type_id
- brand
- model
- serial_number
- ip_address
- mac_address
- remote_address
- os
- unit
- coord_x
- coord_y
- last_seen
- created_at
- updated_at

Table: device_users

- id
- name
- unit

Table: device_user_assignments

- device_id
- user_id

Table: device_connections

- id
- parent_device_id
- child_device_id
- connection_type

Saya ingin membuat API untuk mengelola perangkat sekaligus relasinya.

Gunakan struktur berikut:

Controller:
application/controllers/api/Devices.php

Models:
application/models/Device_model.php
application/models/DeviceUser_model.php
application/models/DeviceUserAssignment_model.php
application/models/DeviceConnection_model.php

Gunakan Query Builder CodeIgniter.

---

API Endpoint yang harus dibuat

1. GET /api/devices

Mengambil semua perangkat beserta:

- daftar user yang menggunakan perangkat
- daftar perangkat yang terhubung

Response contoh:

{
  "status": "success",
  "message": "devices retrieved",
  "data": [
    {
      "id": 7,
      "device_name": "IGD-Dokter",
      "brand": "Acer",
      "model": "Aspire C227-1751",
      "unit": "IGD",
      "users": [
        {
          "id": 2,
          "name": "Dr. Budi"
        }
      ],
      "connections": [
        {
          "device_id": 11,
          "device_name": "Printer IGD"
        }
      ]
    }
  ]
}

---

2. GET /api/devices/{id}

Mengambil detail perangkat beserta:

- device_users
- device_connections

---

3. POST /api/devices

Menambahkan perangkat baru beserta relasi.

Body request contoh:

{
  "device_name": "IGD-Dokter",
  "device_type_id": 1,
  "brand": "Acer",
  "model": "Aspire C227-1751",
  "serial_number": "SN123456",
  "ip_address": "192.168.1.20",
  "mac_address": "00:1B:44:11:3A:B7",
  "os": "Windows 10",
  "unit": "IGD",
  "coord_x": 340,
  "coord_y": 210,
  "users": [2,3],
  "connections": [
    {
      "child_device_id": 11,
      "connection_type": "usb"
    }
  ]
}

Proses:

1. insert ke table devices
2. insert user assignments ke device_user_assignments
3. insert relasi ke device_connections

created_at harus diisi otomatis.

---

4. PUT /api/devices/{id}

Update data device serta relasi:

- update devices
- update device_user_assignments
- update device_connections

updated_at harus diisi dengan datetime saat update.

---

5. DELETE /api/devices/{id}

Menghapus perangkat dan relasi terkait:

- device_user_assignments
- device_connections

---

Model yang harus dibuat

Device_model

fungsi:

getAll()
getById($id)
create($data)
update($id,$data)
delete($id)

DeviceUser_model

getAll()
getById($id)

DeviceUserAssignment_model

getUsersByDevice($device_id)
assignUser($device_id,$user_id)
deleteByDevice($device_id)

DeviceConnection_model

getConnections($device_id)
createConnection($data)
deleteByDevice($device_id)

---

Tambahkan routing di:

application/config/routes.php

$route['api/devices'] = 'api/devices/index';
$route['api/devices/(:num)'] = 'api/devices/show/$1';

---

Gunakan response JSON standar:

Success

{
  "status": "success",
  "message": "text",
  "data": {}
}

Error

{
  "status": "error",
  "message": "text"
}

Pastikan kode kompatibel dengan CodeIgniter 3.1.10 dan dapat langsung diuji menggunakan Postman.
